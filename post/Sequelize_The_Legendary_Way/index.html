Using Sequelize to build a simple node app

In this post, Lets build an awesome todo app for our legendary comic Heroes.

What is Sequelize:

Sequelize is a promise-based ORM for Node.js and io.js. It supports the dialects PostgreSQL, MySQL, MariaDB, SQLite and MSSQL and features solid transaction support, relations, read replication and more.
Before we being, I would recommend you to have a basic understanding of

  1. JavaScript, though I haven’t used much of ES6 constructs, but its good to know.
  2. NodeJs, How the ecosystem works.
  3. What is an ORM, well obviously.
  4. Sequelize, Since this is particularly not an introductory post. I would recommend the following posts.
    • Getting Started with Sequelize and PostgreSQL
    • Getting Started with Sequelize for Nodejs Applications
    • Object-Relational Mapping in Node.js with Sequelize

  • Clone the repo: 
    git clone https://github.com/shivambarsaley/Sequelize_101.git

    git checkout starting_points

    node_modules/.bin/sequelize db:migrate

2. Migrations and Seeders:

Sequelize comes in with a very well build in support for migrations, These are basically managed via the sequelize-cli. It can be installed via npm as 

$ npm install --save sequelize-cli

2.1 Creating New Migrations:

Lets create a new migration to add an ending date to our todo’s, using the sequelize-cli we have

$ node_modules/.bin/sequelize migration:create --name=Adding_End_date_to_Todo

This would create new migration at /server/migrations
Lets update this migration file

# 20170406200830-Adding_End_date_to_Todo.js
'use strict';

module.exports = {
  up: function (queryInterface, Sequelize) {
    
    return queryInterface.addColumn('Todos', 'endDate',
      {
        type: Sequelize.DATE,
        allowNull: true,
      }
    );
  },

  down: function (queryInterface, Sequelize) {
    
    return queryInterface.removeColumn('Todos', 'endDate');
  }
};

And execute the migrate command 

$ node_modules/.bin/sequelize db:migrate

Okay Now that we have the basic structure up, Lets on-board some users, Luckily Steven Rogers and Bruce Banner has signed up to be our early users.

2.2 Lets Add some Seed Data:

Just like migrations, we can create some seeders to populate the Database. Lets start by adding some categories

To create a seed file: 

$ node_modules/.bin/sequelize seed:create --name=Adding_Basic_Categories
This would create new migration at /server/seeders/
Lets seed some data

# 20170406202530-Adding_Basic_Categories.js
'use strict';

module.exports = {

        updatedAt: Sequelize.fn('NOW')

      {
        id: 2,

        id: 3,

        updatedAt: Sequelize.fn('NOW')

Running the seeder task 

`node_modules/.bin/sequelize db:seed:all`

There are couple of more seeders we have for app available here

Unlike migrations, Sequelize does not keep track of the seeders it has run by default, We specifically updated our config to achieve this. Our config now looks like:

The key seederStorage does this for us, So now we are storing it in a table called SequelizeData. We could also have saved it as a json, more on it here.

3. Querying:

Since most of the queries around simple CRUD operations are covered in the introductory tutorials, I ll just leave the samples below:

#server/src/todo.js  

'use strict';
const Model = require('../models/index');

const getTodosForUser = (userId, includeTasks = true) => {
  const options = {
    where: {userId},
    include: [Model.Task]
  };
  if(!includeTasks){
    delete options.include;
  }
  return Model.Todo.findAll(options);
};

const addTodo = (UserId, CategoryId, todoName) => {

  return Model.Todo.create({UserId, CategoryId, name:todoName});
};

const markAsComplete = (id) => {

  return Model.Todo.update({isCompleted: true}, {where:{id}});
};

const deleteTodo = (id) => {

  return Model.Todo.destroy({where:{id}})
};


module.exports = {getTodosForUser, addTodo,markAsComplete, deleteTodo};

3.1 Associations:

Bruce being a avid user, Needs to get more grained data about his todo. Lets get on to it, can’t risk him getting angry.

He wants us to find all his todo’s with just the incomplete tasks, This could be achieved easily by updating the include clause, Lets add a new method to the todo helper.

const getAllTodoWithIncompleteTasksOnly = (UserId) => {

  return Model.Todo.findAll({
    where: {UserId},
    include:[{model: Model.Task, where:{isCompleted: false}}]
  });
}

This should execute the SQL query with an ON clause to just return the incomplete tasks only, Something like

SELECT `Todo`.`id`, `Todo`.`name`, `Todo`.`isCompleted`, `Todo`.`createdAt`, `Todo`.`updatedAt`, `Todo`.`UserId`, `Todo`.`CategoryId`, `Tasks`.`id` AS `Tasks.id`, `Tasks`.`name` AS `Tasks.name`, `Tasks`.`isCompleted` AS `Tasks.isCompleted`, `Tasks`.`createdAt` AS `Tasks.createdAt`, `Tasks`.`updatedAt` AS `Tasks.updatedAt`, `Tasks`.`TodoId` AS `Tasks.TodoId` FROM `Todos` AS `Todo` INNER JOIN `Tasks` AS `Tasks` ON `Todo`.`id` = `Tasks`.`TodoId` AND `Tasks`.`isCompleted` = false WHERE `Todo`.`UserId` = 1;

We could similarly update the include clause to just select some columns say name

include:[{model: Model.Task, where:{isCompleted: false}}, attributes:['name']]

Or to exclude a particular column

include:[{model: Model.Task, where:{isCompleted: false}, attributes: {exclude:['isCompleted']}}]

3.2 Raw Queries:

Often there is a need to run already prepared SQL queries, which can be achieved using the sequelize.query function.

So to run a query

select * from Categories

We can do

#server/src/category.js

const getAllCategoriesRaw = () => {

  Model.sequelize.query('Select * from Categories');
};

Now this would return something like

The structure of response is basically [result, metaData], Now metaData is dialect dependent, and in case of MySQL the response is basically two references to the same result object.

We could get rid of the metadata, by passing the query type. This basically tells sequelize about how to format the result. This could be done as

Model.sequelize.query('Select * from Categories', {type: Model.sequelize.QueryTypes.SELECT});

We can also provide dynamic parameters (stuff to be replaced during query generation) to the raw query, There are two ways to do it

As an array:
#server/src/category.js

const getAllCategoriesReplacementsUsingArray = () => {

  Model.sequelize.query('Select * from Categories where id > ? and id < ?',
  {replacements:[1,4], type: Model.sequelize.QueryTypes.SELECT});
}

Sequelize replaces the ? with the array elements in the order in which they occur.

As reference object
const getAllCategoriesReplacementsUsingObject = () => {

  Model.sequelize.query('Select * from Categories where id > :floorLimit and id < :ceilLimit',
  {replacements:{floorLimit: 1, ceilLimit:4}, type: Model.sequelize.QueryTypes.SELECT});
}

Both of these would generate the same query as

Select * from Categories where id > 1 and id < 4
Stored Procedures:

Since raw queries provide an interface to write direct SQL queries, We could use it to call Stored Procedures, For an example, We have a very minimalistic Stored Proc at /storedProcs called GetAllProducts.

We could use it, as

  Model.sequelize.query('Call GetAllProducts(:id)', {replacements:{id:1}})

3.3 Tiny Little things:

3.3.1 raw: true

Sequelize always returns an instance object of the result, This basically includes functions to perform further operations over the result object. At times, We are not in need for any further operations over the result and only need to read/display.

This flag comes in handy for such cases, It reduces all the overhead sequelize does after fetching.

  Model.Todo.findById(1, {raw:true})

This would give out a result as

{ id: 1,
  name: 'Investigate',
  isCompleted: 0,
  createdAt: 2017-04-16T20:29:02.000Z,
  updatedAt: 2017-04-16T20:29:02.000Z,
  UserId: 1,
  CategoryId: 1 }
3.3.2 Nested Keys

Lets start by adding a new Todo and lets not add any tasks for it. Now we need to count all the todos which don’t have any tasks.This would require accessing a column from the included Tasks table.

Model.Todo.count({include:[{model:Model.Task,required:false}],raw:true,where:{"$Tasks.TodoId$":null}}))

Adding the nested column name in between $ signs would do the trick.

3.3.3 Validation:

Sequelize supports adding validations to models. These validations would be trigged before saving. Lets update our Todo model so name cannot be shorter than 2 and greater than 20.

sequelize.define('Todo', {
    name: { allowNull: false, type: DataTypes.STRING, validate:{len:[2,20]} },
    isCompleted: { type: DataTypes.BOOLEAN, defaultValue: false },
  }

Setting the validate key on name does the trick. Though validations are run before create,update and save, but not before bulkCreate. So even with the validations in place, this

Model.Todo.bulkCreate([{name:"A",CategoryId:1,UserId:1}])

would insert the record in Todos. To enable validations, just specify validate: true as options to bulkCreate,So

Model.Todo.bulkCreate([{name:"A",CategoryId:1,UserId:1}],{validate:true})

would raise an error

3.3.4 having Clause:

Lets try finding the Todo’s having more than 2 tasks. A very simple SQL query for it can be

Select TodoId from Tasks group by TodoId having count(*) > 2;

Sequelize does provide an option to add the having flag. We could simply do:

Model.Task.findAll({group:['TodoId'],attributes:['TodoId'],having:['Count (*) > ?',2]})
3.3.5 before/after keys in migrations:

While adding columns via migrations, the options object supports two flags before/after for ordering of columns.

queryInterface.addColumn('Todos', 'endDate',
  {
    type: Sequelize.DATE,
    allowNull: true,
    after:'CategoryId
    //before: 'createdAt' // Could use a before flag as well
  }
);

Column order do have a performance impact, especially in mysql.

4. Gotcha’s:

4.1 Promise bluebird:

Sequelize provides lots of options for error handling, You can find them here. Lets work upon catching the validation errors, We will update the addTodo method.

const addTodo = (UserId, CategoryId, todoName) => {

  return Model.Todo.create({UserId, CategoryId, name:todoName})
  .catch(Model.Sequelize.ValidationError, (e)=> console.log("Dude... Those fields ain't valid",e))
  .catch((e)=> {console.log('Something else wrong here',e)})
};

The catch block for Validation isn’t the standard JS catch block, Its a filtered catch block which bluebird provides, since sequelize return bluebird promises.

Now this works all well, until we do something like

Promise.all([Model.Todo.create({UserId:1,CategoryId:1,name:"A"})])
.catch(Model.Sequelize.ValidationError, (e)=> console.log("Dude... Those fields ain't valid",e))
.catch((e)=> {console.log('Something else wrong here',e)})

This pops out an error,

> Something else wrong here TypeError: Cannot set property 'name' of undefined
    at error.BaseError (/Users/shivambarsaley/Desktop/website/projects/Sequelize_The_Legendary_Way/node_modules/sequelize/lib/errors.js:22:24)
    at error.ValidationError (/Users/shivambarsaley/Desktop/website/projects/Sequelize_The_Legendary_Way/node_modules/sequelize/lib/errors.js:42:19)

Not something we were expecting, ðŸ˜•.
Since native JavaScript promises do not support filtered catch block, Upon a rejection, it ends in the first catch block and executes Model.Sequelize.ValidationError which again throws another error which which we see upon being caught in the last catch block. We can solve this by using the standard catch block and instanceof operator.

Promise.all([Model.Todo.create({UserId:1,CategoryId:1,name:"A"})])
.catch((e) => {
  if(e instanceof Model.Sequelize.ValidationError){
    console.log('Yipee')
  }
});

or by using Promise from sequelize, like

 Model.Sequelize.Promise.all([
  Model.Todo.create({UserId:1,CategoryId:1,name:"A"})
])
.catch(Model.Sequelize.ValidationError, (e)=> console.log("Dude... Those fields ain't valid",e))
.catch((e)=> {console.log('Something else wrong here',e)});

or by using requiring bluebird.

const Promise = require('bluebird');
Promise.all([Model.Todo.create({UserId:1,CategoryId:1,name:"A"})])
.catch(Model.Sequelize.ValidationError, (e)=> console.log("Dude... Those fields ain't valid",e))
.catch((e)=> {console.log('Something else wrong here',e)});

4.2 Adding Foreign Keys:

If you are using a version of sequelize < 4.0. There is a high possibility you might run into this issue, just like I did. To start with you should upgrade to v4, and use the addConstraint method.

Unfortunately, this wasn’t so easy for us. So sticking to Sequelize 3.x, There is an ugly way of using raw queries in migrations So something like

queryInterface.sequelize.query('ALTER TABLE `Teachers` ADD CONSTRAINT Teachers_Subject_id FOREIGN KEY (SubjectId) REFERENCES Subjects(id);')

post creating the SubjectId column.

Summarizing:

Sequelize is an extensive and powerful ORM for NodeJs. Though I didn’t get a chance to work across other Javascript ORM’s but Sequelize does seem to have a leading edge against others.
